# KJXJ Cloud Drive Sync

将请求分为“只读审核建议”和“确认后执行”两个阶段。除非用户已审核本次建议并明确确认，不得改变云盘、来源文件或生成压缩副本。

## 路径与规则

- 优先使用全局说明中定义的云盘根目录及规则文件；否则使用：
  - 根目录：`/Users/apple/Documents/跨境电商/google-driver/`
  - 规则文件：`/Users/apple/Documents/跨境电商/google-driver/跨境电商云盘目录方案_v1.md`
- 每次处理前阅读规则文件中的目录规则和“资源索引”，并检查与建议目标有关的目录和文件。资源索引是长期复用资源的定位来源，命中时优先建议更新既有资源，避免重复创建。
- 使用 `<skill-dir>/scripts/check_sync_item.py` 进行文件只读预检；使用 `find_sync_candidates.py` 搜索相似文件或可复用台账。两者均不得写入。

## 不可违反的规则

- 在本次提案后得到明确确认前，不得复制、覆盖、创建目录或文件、修改工作簿、压缩文件、删除、重命名或写入日志。
- 单个最终文件或文本载荷不得超过 10 MiB。不得删除来源文件、云盘历史文件或既有日志。
- 不得将普通业务文件直接写入根目录。低置信度归类使用 `99_待整理`，保留原文件名且不覆盖。
- 只有提案展示了完整目标路径且用户明确确认覆盖时才能覆盖。提案修改后，必须重新等待确认。
- 不得重命名目录管理文档；仅能在其现有文件名下更新目录树、当前状态或资源索引。
- 目录管理文档的资源索引只维护长期复用资源：新建的业务目录、知识库、模板、台账、SOP，或用途发生实质变化的既有长期资源。普通报表、一次性文件和仅递增版本号的文件不得登记。
- 同步结果为成功、跳过、拒绝或失败时，都必须尝试记录操作日志；日志失败必须在结果中明确说明。

## 只读审核阶段

### 1. 解析和归类

识别文件或文本、用户意图、公司、平台、站点、日期、主题、版本和业务类别。按以下优先级建议目标：

1. 跨公司模板、通用 SOP、培训或公共素材：`00_共享中心`。
2. 明确公司：`01_公司档案/公司名`。
3. 依规则文件映射业务类别；周期性报表继续映射到 `08_数据报表/年份/月度`。
4. 无法高置信度判断：`99_待整理`。

遵守规则文件中的文件命名格式 `日期_平台或项目_主题_版本`；提案同时显示原始名称和建议名称。

在归类前，使用资源索引的“内容范围/匹配关键词”和“更新方式”检索既有长期资源。高置信度命中时，提案必须优先显示该资源为建议目标；仅当现有资源无法承载内容、用途不同或匹配不确定时，才能建议新建或要求用户选择。

### 2. 文件预检和冲突处理

对已知来源文件执行：

```bash
python3 <skill-dir>/scripts/check_sync_item.py --source "/absolute/source" --target "/absolute/target" --hash
```

- 目标不存在：建议 `新增`。
- 哈希相同：建议 `跳过`。
- 目标存在但未明确覆盖：保留旧版，建议递增版本号新建；不可自行覆盖。
- 来源超过 10 MiB：仅限图片/PDF 可建议压缩；其他类型拒绝同步。
- 图片/PDF 压缩只能在确认后通过 `compress_media.py` 输出到临时明确路径。预检阶段不得实际压缩。PDF 环境无可用压缩器时，如实说明不可压缩。

### 3. 文本台账决策

先提取结构化字段，搜索已分类目录中的 `.xlsx`、`.xls`、`.csv`、`.tsv` 候选台账，并阅读候选表头。

- 存在主题与字段均匹配的台账：建议写入现有 `数据` 工作表或其实际数据工作表，优先沿用已有列。
- 不存在匹配台账：建议在归类目录新建 `主题信息台账.xlsx`，使用 `数据` 工作表；列为提取字段，另加 `来源文本`、`录入日期`。
- 完全相同的唯一实体：建议跳过。
- 唯一实体高置信度匹配且仅需补充/修正：建议更新该行，展示受影响字段。
- 关键字段冲突或多个候选：标记 `需用户选择`，不得写入。
- 用公司名称加电话/邮箱/地址、或店铺加 ASIN/SKU 等实际可得唯一字段作匹配键；在提案中明确匹配键。

### 4. 资源索引决策

目录管理文档中的“资源索引”采用以下字段：`资源路径`、`资源类型`、`用途`、`内容范围/匹配关键词`、`更新方式`、`当前状态`。

- 新建长期业务目录时，登记目录路径和用途；目录内已有明确长期载体时，可另行登记该载体，避免用目录条目替代具体更新目标。
- 新建知识库、模板、台账或 SOP 时，登记其文件路径、用途、可匹配的高层主题关键词和追加/更新方式。
- 向既有知识库追加碎片内容时，除非知识库用途、覆盖范围或更新方式改变，不记录单个知识点，也不更新资源索引。
- 仅更新资源索引时，先精确匹配资源路径；路径相同则修订该行，禁止新增重复条目。
- 无法判断资源是否长期复用或无法确定索引摘要时，提案中标记“需用户选择”，不得自行更新目录管理文档。

## 提案格式

始终先输出以下信息并停止等待确认：

```markdown
## 云盘同步建议

- 同步类型：文件 / 文本台账
- 规则文件：
- 归类理由：
- 源内容：路径、大小、哈希；或解析字段
- 建议目标：完整绝对路径
- 操作建议：新增 / 新增记录 / 更新已有记录 / 跳过 / 覆盖 / 需用户选择
- 命名与冲突情况：
- 大小与压缩：
- 目录管理文档更新：不更新 / 新增资源索引 / 更新资源索引；展示完整管理文档路径、目标章节、拟增改摘要，并注明文件名保持不变
- 日志文件：`98_同步日志/YYYY-MM_云盘同步操作日志.xlsx`
- 执行前确认：

确认后我将执行：
1. ...
2. ...
3. 按提案更新并核验目录管理文档的资源索引（如适用）。
4. 写入并核验月度日志。
```

用户只有使用“确认执行”“按这个执行”“可以执行”等明确语义确认后，才可进入执行阶段。

## 确认后执行与验证

1. 再次检查目标冲突和大小，避免审核后状态变化。
2. 必要时调用 `compress_media.py`，验证压缩产物小于等于 10 MiB、图像可读取；PDF 保留输出文件且压缩器成功返回。失败或仍超限则不写入云盘。
3. 复制或替换文件时使用建议中的精确路径；复制后比较大小，必要时比较哈希。
4. 若提案包含资源索引更新，在主同步成功后更新原目录管理文档的“资源索引”及必要目录树/当前状态；先写入临时文件，核验资源路径、摘要、文件名不变且无重复条目后再替换。不得创建或改名第二份目录管理文档。
5. 文本台账和日志使用 `ledger_ops.mjs`。在临时工作目录创建到 Codex 依赖 `node_modules` 的符号链接后执行，使用显式 `--input` 与 `--output`，并在替换目标前验证输出文件大小。仅向用户已确认的目标写入。
6. 重新读取目标文件、台账和目录管理文档，确认文件存在、行内容或预期字段已出现，且资源索引的实际路径可访问。

## 月度同步日志

- 日志目录固定为 `<cloud-root>/98_同步日志`；仅在首次确认后的执行阶段创建。
- 日志文件按自然月拆分：`YYYY-MM_云盘同步操作日志.xlsx`，工作表名称为 `操作日志`。
- 每项最终操作追加一行：`操作时间`、`操作用户标识`、`操作内容摘要`、`同步类型`、`最终动作`、`源路径/来源`、`目标路径`、`结果`、`备注`。
- 操作用户标识优先为 `device-<hash>`：对本机硬件唯一标识进行 SHA-256 后取短前缀；不得写入原始硬件 ID。无法取得时记录本机账户并在备注写明身份来源。
- 摘要必须包含归类理由，以及版本、压缩、台账字段变更或资源索引变更等结果。日志文件超过 10 MiB 前应建议按月内分卷（`YYYY-MM_云盘同步操作日志_02.xlsx`），不得超限写入。

## 脚本接口

- `check_sync_item.py`：只读检查文件存在性、大小、MIME、可压缩性、哈希和目标冲突。
- `find_sync_candidates.py --root <root> --query <keywords> --mode files|workbooks`：只读返回排序候选项。
- `get_operator_id.py`：返回匿名 `device-<hash>` 操作用户标识；仅在无法读取硬件标识时回退至本机账户。
- `compress_media.py --source <source> --output <temp-output>`：仅确认后创建压缩副本，绝不修改来源。
- `ledger_ops.mjs --mode ledger|log --input <existing-or-omit> --output <target> --record <json> --operation append|update|skip [--sheet <name>]`：确认后新建或更新台账/日志，并验证输出小于等于 10 MiB。

## 失败处理

- 规则文件或云盘根目录不可读：停止并报告，不能猜测路径或创建根目录。
- 找不到来源文件：报告查找结果并要求绝对路径。
- 多个候选台账、关键字段冲突、或覆盖意图不明确：展示候选并等待选择。
- 资源索引与实际路径不一致、同一资源存在多个索引候选、或资源用途无法确定：展示冲突并等待选择，不得擅自创建或更新索引条目。
- 主同步成功但目录管理文档更新失败：记录“主同步已完成但资源索引更新失败”及原因，并在月度日志中写入部分成功；不得声称整个流程已完全成功。
- 主操作完成但日志失败：报告“同步结果已完成但日志失败”及失败原因；不得声称整个流程已完全成功。
